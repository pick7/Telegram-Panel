# 常见问题

## 代理页显示“未使用 WARP”，是不是独立 WARP 失效了

不一定。代理页顶部检测的是面板服务自身出口；“未使用 WARP”只表示面板本身没有走
Cloudflare WARP。下方代理列表会单独显示每个 WARP 容器的出口和“WARP 已连接”状态。

## 出口 IP 是一串带冒号的字符，为什么没有城市和运营商

带冒号的地址通常是有效 IPv6。当前检测源稳定提供 IP、国家码和 WARP 状态，不保证返回
城市或运营商；这些字段为空不表示检测失败。详情见
[代理管理与账号出口](../guides/proxy-management.md)。

## WARP 容器还在运行但出口挂了怎么办

受管 WARP 默认会自动巡检。连续两次出口检测失败后，面板会重启原容器、保留数据卷并
重新检测；绑定账号的客户端会重新建立，但不会降级为直连。代理页会显示连续失败次数、
恢复状态、最近与下次巡检时间，也可以手动刷新单个或全部 WARP。

健康 WARP 默认不会每 720 分钟强制重启，以免无故更换 Telegram 账号 IP；确需参考
tokens-pro 的周期刷新时，可设置 `TP_WARP_SCHEDULED_REFRESH_ENABLED=true`。

## 发送验证码提示“session 被占用/进程占用”

- 确保不要多开多个面板实例共享同一个 `sessions` 目录（会互相锁文件）
- 如果刚修改了全局 `ApiId/ApiHash`，建议重启面板再登录
