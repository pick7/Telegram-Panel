# TPM 发布与生产验收

按下面顺序执行门禁。任一层失败都停止部署，不用后一层结果掩盖前一层问题。

源码、TPM 内容、干净宿主安装和生产复核适用于所有模块。其余门禁按能力执行：

- 有管理页面：执行 HTTP 与浏览器验收。
- 提供 API：执行鉴权、输入和响应合同验收。
- 含 HostedService/长任务：执行后台真实业务验收。
- 不具备某项能力：在发布证据中标记“不适用”及原因，不伪造通过结果。

## 1. 源码与测试

- 确认目标宿主 Tag/提交与模块编译基线一致，工作树无意外修改。
- 运行模块单元测试、静态检查和 Release 编译；单个后台测试设置不超过 60 秒的
  超时。
- 检查源码 Manifest、根 `manifest.json`、CI 包名及测试断言的版本一致。
- 使用新宿主 ABI 时，提高 `host.min` 并增加目标宿主兼容测试。

## 2. TPM 内容

- 文件名严格为 `<moduleId>-<version>.tpm`。
- 根目录包含 `manifest.json`，`lib/` 包含入口 DLL。
- 静态页面模块包含 HTML、JS、CSS 及本地 Vue 运行时。
- 不包含宿主共享程序集、历史 `publish/`、测试产物、凭据或临时文件。
- 以无 BOM UTF-8 生成 Manifest。迁移工具读取第三方旧包时应能识别 BOM 并给出
  明确诊断，但正式发布门禁默认拒绝 BOM；不得让部署器以普通 UTF-8 解析后静默失败。
- 若包中存在第二份 Manifest，要求与根 Manifest 语义一致；否则删除副本。
- 对 TPM 运行仓库统一校验器；本地、CI 和部署工作流不得各自实现不同规则。

## 3. 干净宿主安装

1. 使用与生产完全相同的宿主镜像/二进制和配置启动临时宿主。
2. 从空模块目录安装 TPM，启用模块并重启宿主。
3. 确认模块 `Enabled=true`、`ActiveVersion` 为新版本，启动日志没有加载异常。
4. 确认 `LastGoodVersion` 在成功加载后指向新版本，且没有静默回滚到旧包。

同版本目录通常不能覆盖安装。修复任何代码或资源后都要递增模块版本，禁止靠
手工替换已安装 DLL 完成发布。

## 4. 页面模块：HTTP 与浏览器烟雾测试

- 探测页面 HTML、每个 JS/CSS/字体、聚合管理 API，要求预期状态码和
  `Content-Type`。
- 从侧栏进入 `/ui/ext/...`，确认页面在当前后台 iframe 中渲染。
- 使用真实浏览器检查可见内容、控制台异常、失败请求、无限重定向和混合内容。
- 执行一次读取和保存，确认局部更新及刷新后持久化。
- 不把“HTML 返回 200”视为页面通过；空白页面、资源 404 或 API 500 都必须失败。

## 5. API 模块：接口合同验收

- 验证匿名/管理员鉴权选择与设计一致，未授权请求不能回退成 HTTP 200 登录页。
- 覆盖合法请求、缺少参数、非法参数和业务失败，核对状态码、`Content-Type` 与 DTO。
- 至少完成一次真实调用；具有外部副作用时使用可控测试数据并保留清理或回滚证据。

## 6. 后台模块：业务验收

- 对 HostedService/长任务执行一次“立即运行”或等待一轮完整执行。
- 验证开始/结束时间、心跳、Summary、错误和业务变化时间语义正确。
- 根据配置断言预期来源已检查，`Errors=0`、无意外跳过，并验证至少一个可控的
  真实业务结果；没有新数据时也要证明检查范围完整。
- 模拟一次可恢复失败，确认原始错误、退避时间和下一次成功恢复可见。

## 7. 生产部署与回滚

1. 部署前保留当前 TPM、安装目录、状态文件和数据库备份。
2. 使用与现网相同的 Compose 文件集合、环境文件、卷和网络重建服务；例如启用
   WARP 时不能漏掉对应 override。
3. 重启后复核宿主版本、模块 `Enabled/ActiveVersion/LastGoodVersion`、日志、页面、
   API 和后台业务。
4. 若加载或业务验收失败，切回已验证的旧 `ActiveVersion` 并重启；不要删除
   `LastGoodVersion`，直到新版本稳定通过。

## 发布证据

保留以下可追溯结果：

- 宿主版本、Tag、提交和镜像摘要。
- 模块版本、TPM 路径及 SHA-256。
- 校验器、测试、安装和重启结果，以及适用的页面/API 验收结果。
- 后台模块的一轮真实 Summary；不含后台能力时标记不适用。
- 生产部署时间、当前版本、已知良好版本和回滚结果（如发生）。

只有公共门禁与该模块实际具备能力对应的门禁全部通过，才能宣告模块发布完成。
